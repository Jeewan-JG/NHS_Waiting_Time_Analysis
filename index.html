NHS Waiting Times Analysis

Volume, Delays & Provider Performance Analysis using Python and Tableau.

Project Overview

This project analyses NHS waiting list data to understand the scale of long waiting times, identify the drivers of the 52+ week backlog, and compare provider performance. The goal is to uncover actionable insights that can support capacity planning, improve patient outcomes, and reduce long waiting times.

The analysis focuses on:

  • Total patient volume and long-wait backlog
  • The percentage of patients waiting over 52 weeks
  • Monthly changes in waiting performance
  • Differences in provider performance
  • The relationship between patient volume and waiting times
  • The distribution of long waiting times across week bands

Business Problem

NHS providers are under pressure from long patient waiting times, especially for patients waiting more than 52 weeks for treatment.

This project aims to answer:

  • How severe is the 52+ week backlog?
  • Are waiting times improving or worsening over time?
  • Which providers handle the highest patient volumes?
  • Which providers have the highest 52+ week wait ratios?
  • Do higher patient volumes lead to longer waits?

Why this matters

  • Long waits can worsen patient outcomes and increase operational pressure.
  • High-performing and low-performing providers can be compared to identify gaps.
  • Leaders need clear evidence to decide where to increase capacity and focus support.

📂 Dataset

The dataset contains NHS waiting list data with fields such as:

  • Provider
  • Period Date
  • Total Patients
  • Wait 52 Plus
  • Wait Ratio KPI
  • MoM % Change Wait 52 Plus
  • Gt 52 To 53 Weeks SUM 1 through Gt 99 To 100 Weeks SUM 1

These week-band columns were especially important because they allowed the project to measure the long-tail backlog and build the waiting time distribution chart.

Data Cleaning in Python

Python was used to combine multiple monthly NHS files, clean the raw dataset, handle duplicates and missing values, convert numeric fields, and create the key 52+ week waiting metric used throughout the dashboard.

1. Load a single file and inspect the raw structure

import pandas as pd

file_path = "/Users/jeewangurung/Downloads/NHS_case_study/20250331-RTT-March-2025-full-extract-revised.csv"
df = pd.read_csv(file_path)

2. Combine all monthly CSV files into one dataset

import os

folder = "/Users/jeewangurung/Downloads/NHS_case_study"

all_files = [os.path.join(folder, f) for f in os.listdir(folder) if f.endswith(".csv")]
df_list = [pd.read_csv(file) for file in all_files]

combined_df = pd.concat(df_list, ignore_index=True)

combined_df.shape
combined_df.to_csv("/Users/jeewangurung/Downloads/nhs_combined_4months.csv", index=False)

3. Create a safe working copy and remove duplicates

clean_df = combined_df.copy()

clean_df.duplicated().sum()
clean_df = clean_df.drop_duplicates()

4. Check missing values and convert numeric columns

clean_df.isna().sum().sort_values(ascending=False).head(15)

numeric_cols = [col for col in clean_df.columns if "SUM" in col or col == "Total"]

for col in numeric_cols:
    clean_df[col] = pd.to_numeric(clean_df[col], errors="coerce")

5. Remove invalid rows and keep only useful columns

clean_df = clean_df.dropna(subset=[
    "Period",
    "Provider Org Name",
    "RTT Part Type",
    "Total"
])

clean_df = clean_df[clean_df["Total"] >= 0]

columns_to_keep = [
    "Period",
    "Provider Org Name",
    "Commissioner Org Name",
    "RTT Part Type",
    "Total",
    "Patients with unknown clock start date"
]

waiting_cols = [col for col in clean_df.columns if "Gt " in col and "Weeks SUM 1" in col]

clean_df = clean_df[columns_to_keep + waiting_cols].copy()

6. Rename columns for easier analysis

clean_df = clean_df.rename(columns={
    "Period": "period",
    "Provider Org Name": "provider",
    "Commissioner Org Name": "commissioner",
    "RTT Part Type": "rtt_type",
    "Total": "total_patients",
    "Patients with unknown clock start date": "unknown_clock"
})

7. Fill missing waiting-time values with zero

waiting_cols = [col for col in clean_df.columns if "Gt " in col]
clean_df[waiting_cols] = clean_df[waiting_cols].fillna(0)

8. Select only columns for patients waiting 52+ weeks

long_wait_cols = [
    col for col in clean_df.columns
    if "Weeks" in col
    and col.startswith("Gt ")
    and int(col.split()[1]) >= 52
]

long_wait_cols[:10]

9. Create the main 52+ week waiting metric

clean_df["wait_52_plus"] = clean_df[long_wait_cols].sum(axis=1)

clean_df[["period", "total_patients", "wait_52_plus"]].head()

10. Save the cleaned dataset for Tableau

clean_df.to_csv("/Users/jeewangurung/Downloads/final_nhs_analysis.csv", index=False)

Python Analysis

After cleaning, Python was also used to calculate the main KPIs, build monthly trends, compare providers, and identify high-impact organisations with both high backlog size and high wait ratios.

1. Calculate the overall percentage waiting over 52 weeks

total_patients_all = clean_df["total_patients"].sum()
total_long_wait = clean_df["wait_52_plus"].sum()

percentage_long_wait = (total_long_wait / total_patients_all) * 100

print(f"Percentage of patients waiting over 52 weeks: {percentage_long_wait:.2f}%")

2. Total patient trend by month

trend = clean_df.groupby("period")["total_patients"].sum()
trend

3. 52+ week waiting trend by month

long_wait_trend = clean_df.groupby("period")["wait_52_plus"].sum().reset_index()
long_wait_trend

4. Total patients per provider

provider_totals = clean_df.groupby("provider")["total_patients"].sum().reset_index()
provider_totals.head()

5. Total long waits per provider

provider_long_waits = clean_df.groupby("provider")["wait_52_plus"].sum().reset_index()
provider_long_waits.head()

6. Build provider analysis table with wait ratio

provider_analysis = provider_totals.merge(provider_long_waits, on="provider")

provider_analysis["wait_ratio"] = (
    provider_analysis["wait_52_plus"] / provider_analysis["total_patients"]
)

provider_analysis.head()

7. Identify the worst providers by wait ratio

worst_providers = provider_analysis.sort_values(by="wait_ratio", ascending=False)
worst_providers.head(10)

8. Focus on providers with real operational impact

worst_providers_real_impact = worst_providers[
    worst_providers["wait_52_plus"] > 1000
]

worst_providers_real_impact.head(10)

9. Filter providers with both high volume and high wait ratio

filtered_providers = worst_providers[
    (worst_providers["wait_52_plus"] > 1000) &
    (worst_providers["wait_ratio"] > 0.1)
]

filtered_providers.head(10)

📊 Key Insights

Based on the analysis and dashboard, several important patterns were identified.

1. Backlog remains significant

Around 7.09% of patients, equal to 841,880 people, are waiting over 52 weeks. This shows a major backlog that still needs focused action.

2. Higher volume links to longer waits

The scatter plot suggests that providers with larger patient volumes tend to experience longer waiting times, pointing to capacity pressure.

3. Provider performance is uneven

Some providers have much higher wait ratios than others, showing that performance is not consistent across the system.

4. Progress is unstable

Monthly changes show short-term improvement followed by renewed increases, meaning progress has not yet become stable.

5. Long-tail waiting issue

The waiting time distribution shows that while most patients are concentrated in earlier week bands, a smaller group experiences very long delays, which can be especially harmful.
