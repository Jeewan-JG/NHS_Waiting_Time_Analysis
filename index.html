NHS Waiting Times Analysis: Volume, Delays & Provider Performance

This project analyses NHS waiting list data to understand the scale of the 52+ week backlog, identify the drivers of long delays, and compare provider performance through KPI analysis, trend analysis, provider ranking, and dashboard storytelling.

Business Problem

The NHS is under growing pressure from long patient waiting times, particularly for patients waiting more than 52 weeks. The analysis focused on turning raw data into business questions leaders can act on.

  • How severe is the 52+ week waiting backlog?
  • Do higher patient volumes lead to longer waiting times?
  • Which providers contribute most to poor waiting performance?
  • Are waiting times improving or worsening over time?

Data Preparation

Python was used to clean the dataset, build the 52+ week KPI logic, group data by month, and prepare provider-level analysis tables before dashboard development in Tableau.

  • Cleaned missing and inconsistent values
  • Created wait_52_plus metric from 52+ week columns
  • Built KPI calculations for total patients and wait ratio
  • Prepared monthly trend tables and provider rankings
  • Loaded transformed outputs into Tableau for dashboard design

Interactive Dashboard

The dashboard combines KPI cards, backlog trends, provider rankings, a scatter plot showing the relationship between patient volume and waiting time, month-over-month change, and waiting time distribution analysis.

Python Workflow

Below is the Python logic used to clean the data, create the 52+ week waiting metric, and prepare the KPI and trend outputs that fed into Tableau.

1) Load and prepare the dataframe

import pandas as pd
import numpy as np

# Load the dataset
df = pd.read_csv("final_nhs_analysis.csv")

# Basic cleaning
clean_df = df.copy()
clean_df.columns = [col.strip() for col in clean_df.columns]

# Convert date column
clean_df["Period Date"] = pd.to_datetime(clean_df["Period Date"], errors="coerce")

# Fill missing numeric values
numeric_cols = clean_df.select_dtypes(include=["number"]).columns
clean_df[numeric_cols] = clean_df[numeric_cols].fillna(0)

2) Build the 52+ week metric

# Select all waiting time columns above 52 weeks
gt_52_cols = [
    col for col in clean_df.columns
    if col.startswith("Gt ") and "Weeks SUM 1" in col
]

# Create combined 52+ week waiting metric
clean_df["wait_52_plus"] = clean_df[gt_52_cols].sum(axis=1)

# Keep a simplified total patients field
clean_df["total_patients"] = clean_df["Total Patients"]

3) Create KPI values

total_patients_kpi = clean_df["total_patients"].sum()
wait_52_plus_kpi = clean_df["wait_52_plus"].sum()
wait_ratio_kpi = wait_52_plus_kpi / total_patients_kpi

print("Total Patients:", f"{total_patients_kpi:,.0f}")
print("Patients Waiting 52+ Weeks:", f"{wait_52_plus_kpi:,.0f}")
print("Wait Ratio:", f"{wait_ratio_kpi:.2%}")

4) Monthly trend analysis

monthly_trend = (
    clean_df.groupby(clean_df["Period Date"].dt.to_period("M"))
    .agg(
        total_patients=("total_patients", "sum"),
        wait_52_plus=("wait_52_plus", "sum")
    )
    .reset_index()
)

monthly_trend["wait_ratio"] = (
    monthly_trend["wait_52_plus"] / monthly_trend["total_patients"]
)

monthly_trend["Period Date"] = monthly_trend["Period Date"].astype(str)
print(monthly_trend)

5) Provider-level comparison

provider_summary = (
    clean_df.groupby("Provider")
    .agg(
        total_patients=("total_patients", "sum"),
        wait_52_plus=("wait_52_plus", "sum")
    )
    .reset_index()
)

provider_summary["wait_ratio"] = (
    provider_summary["wait_52_plus"] / provider_summary["total_patients"]
)

top_10_by_patients = provider_summary.nlargest(10, "total_patients")
top_10_by_wait_ratio = provider_summary.nlargest(10, "wait_ratio")

print(top_10_by_patients.head())
print(top_10_by_wait_ratio.head())

6) Month-over-month change

monthly_trend["mom_change_wait_52_plus"] = (
    monthly_trend["wait_52_plus"].pct_change()
)

monthly_trend["mom_change_wait_ratio"] = (
    monthly_trend["wait_ratio"].pct_change()
)

print(
    monthly_trend[
        ["Period Date", "wait_52_plus", "wait_ratio", "mom_change_wait_52_plus"]
    ]
)

Key Insights

  • 7.09% of patients (841,880) are waiting over 52 weeks, showing a significant backlog.
  • Higher patient volumes are linked to longer waiting times, indicating capacity pressure.
  • Performance varies across providers, with some showing much higher wait ratios.
  • Waiting times are unstable, with recent increases after a short improvement.
  • A small number of patients experience very long delays.

Recommendations

  • Redistribute workload across providers to balance capacity.
  • Target high wait-ratio providers for improvement.
  • Improve triaging and expand digital consultation services.
  • Monitor waiting times continuously and respond quickly to increases.
  • Prioritise reducing the 52+ week backlog.

Conclusion

This project demonstrates how healthcare data can be transformed into practical business insight. By combining Python-based data preparation with Tableau dashboard design, the analysis highlights backlog severity, capacity pressure, provider-level variation, and the need for targeted interventions to reduce long waiting times.
